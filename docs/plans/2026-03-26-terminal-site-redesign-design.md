# Terminal Site Redesign Design

## Goal

Redesign the entire `web/` app into a hard-edged intelligence terminal while preserving the product’s defining idea:

> the country itself is the primary object on the site

The featured country for the redesigned homepage should be `Iran`.

## Product Position

This should not look like:
- a soft editorial startup landing page
- a generic AI dashboard
- a glassmorphism metrics wall

It should feel like:
- a disciplined geopolitical intelligence product
- a dark terminal-grade command surface
- a sharp, rectangular, high-signal interface

## Visual System

### Palette

- background: near-black graphite
- panel: dark steel / charcoal
- text: off-white with muted steel secondary text
- alert red: restrained but strong
- amber: secondary emphasis for caution / neutral stress
- green: only for positive directional deltas or comparatively stabilizing signals

### Shape Language

- square or near-square corners
- straight dividers and hard panel boundaries
- no soft floating cards
- no blob gradients
- no rounded hero containers

### Typography

- keep `Inter`
- tighter uppercase metadata
- more compact numeric emphasis
- denser headers and panel labels
- tabular-feeling metrics where useful

### Motion

- minimal
- no bloom halo or soft pulse cloud
- subtle scan, line sweep, or restrained grid emphasis only

## Structural Principle

The country silhouette remains the dominant visual object.

That means:
- the silhouette is larger than charts
- the silhouette is visually stronger than any global map
- the silhouette sits inside a harder shell, not a soft editorial card
- the supporting panels orbit the country object rather than replacing it

## Homepage Direction

### Layout

Use a three-zone desktop layout:
- left rail: compact current-state and driver modules
- center stage: Iran silhouette plus primary score and trend
- right rail: timeline, latest report, watchlist, and supporting metrics

### Content hierarchy

1. Iran silhouette and risk state
2. top-line risk / horizon / confidence / update time
3. current drivers
4. scenario framing
5. timeline and latest reporting
6. secondary global context

### What changes from the current site

- fewer soft open spaces
- more rectilinear framing
- darker shell
- denser signal presentation
- stronger use of grid, separators, and alignment

## Forecast Explorer Direction

The forecast explorer should look closer to a ranked operations console than a marketing table.

Required traits:
- rigid table grid
- strong header strip
- compact filters
- dense numeric columns
- more visually explicit deltas
- minimal decoration

It should still remain readable and credible, not theatrical.

## Country Page Direction

Country pages should become intelligence dossiers.

Recommended pattern:
- hard hero block with silhouette, score, confidence, freshness
- side-by-side signal panels
- rectangular scenario panels
- tighter trend and timeline modules
- report archive and related-country panels in a clear right or lower rail

## Reports and Methodology Direction

Reports should remain readable long-form pages, but inside the same terminal shell:
- dark background
- hard section dividers
- tighter metadata rows
- restrained callouts
- no blog-style airy card stacks

Methodology should use the same system, but quieter:
- stronger grid
- calmer signal colors
- less emphasis on alerts

## Iran Requirement

The redesign must include:
- `iran` in the country shape system
- `iran` as the featured homepage country
- Iran-specific sample/report/demo data replacing Sudan as the default hero case

## Component Consequences

This redesign implies changes to:
- global theme tokens
- header / footer shell
- badge system
- hero silhouette component
- explorer table styling
- report cards
- scenario panels
- timeline panels
- methodology callouts

The current rounded editorial component language should be treated as a replaceable first pass, not a stable design system.

## Scheduling Rule

This redesign is approved as the target site-wide direction.

But implementation should remain downstream of:
1. complete data layer
2. real-data forecasting
3. backtesting

unless project priority changes explicitly.
